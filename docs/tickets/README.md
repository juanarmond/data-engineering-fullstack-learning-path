# Tickets Guide

Use tickets as the day-by-day execution plan.

## How To Navigate

Follow this hierarchy:
1. `../../README.md` (project-level)
2. `../roadmap.md` (phase-level)
3. `milestones/<milestone>/README.md` (milestone-level)
4. `milestones/<milestone>/week-<nn>/README.md` (week-level)
5. day tickets (execution-level)

## Milestone Entry Points

- `docs/tickets/milestones/m0/README.md`
- `docs/tickets/milestones/m1/README.md`
- `docs/tickets/milestones/m2/README.md`

## Recommended Execution Path

1. `milestones/m0/week-01/`
2. `milestones/m1/week-01/`
3. `milestones/m1/week-02/`
4. `milestones/m2/week-01/`

## Ticket Rules

A good ticket must include:
- goal,
- context,
- tasks,
- acceptance criteria,
- verification steps.

## Weekly Cadence

At week end:
- mark completed tickets,
- list blockers,
- confirm deliverables,
- write handoff notes,
- then move forward.
