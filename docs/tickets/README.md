# Tickets Guide

This folder is the execution guide for the learning journey.
Use tickets as your day-by-day plan.

## What This Guides

Tickets guide you through the project in this order:
1. onboarding fundamentals,
2. milestone work,
3. weekly progression toward a working dashboard.

Each ticket should represent one focused day of work.

## Ticket Structure

Tickets are organized as:
- `docs/tickets/milestones/<milestone>/week-<nn>/`

Each week folder should include:
- a `README.md` summary of what will be learned in that week,
- daily ticket files for that week.

Examples:
- `docs/tickets/milestones/m0/week-01/`
- `docs/tickets/milestones/m1/week-01/`
- `docs/tickets/milestones/m2/week-01/`

## How To Use Tickets

1. Start at onboarding (`m0`).
2. Read that week's `README.md` first.
3. Complete tickets in order (Day 1 -> Day 7).
4. Do only the scope inside the current ticket.
5. Record blockers and notes before moving to the next day.
6. At week end, review progress and plan the next week.

## Ticket Quality Rules

A good ticket must include:
- clear goal,
- short context,
- concrete tasks,
- acceptance criteria,
- verification steps.

## Weekly Cadence

At the end of each week:
- mark completed tickets,
- list what was blocked,
- update the next week's tickets.
