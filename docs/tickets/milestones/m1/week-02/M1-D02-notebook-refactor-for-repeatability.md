# Ticket: Notebook Refactor for Repeatability

## Milestone
M1

## Day
D02

## Goal
Refactor notebook structure to be readable and repeatable.

## Context
A clear notebook layout reduces mistakes and improves handoff quality.

## Tasks
- Organize notebook sections (load, profile, quality checks, findings).
- Move reusable constants (paths, required columns, thresholds) to a single config cell.
- Add markdown headings and short interpretation notes.
- Ensure every output can be regenerated by rerunning cells.
- Use explicit section headers that mirror Week 2 flow (`Reproducibility`, `Config`, `Profile Metrics`, `Quality Checks`, `Report Summary`, `Runbook`, `Weekly Review`).

## Acceptance Criteria
- Notebook has a clear sectioned structure.
- Config values are centralized.
- Another learner can follow the logic without verbal explanation.
- Section headings are visible and consistently used from top to bottom.

## Verification
- Restart kernel and run all cells.
- Manual review for section clarity.

## Notes
Keep this focused on structure and readability, not new business logic.
