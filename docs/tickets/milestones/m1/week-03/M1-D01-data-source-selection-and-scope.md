# Ticket: Data Source Selection and Scope

## Milestone
M1

## Day
D01

## Goal
Choose the real dataset you will carry through the rest of the learning path and define a realistic initial project scope.

## Context
After onboarding, the learner should start with real data and limit scope to avoid overload.
This dataset becomes the project dataset for the rest of M1 and for the M2 ETL work.

## Tasks
- Review the suggested source: `https://www.football-data.co.uk/data.php`.
- Choose one league and one season for your project dataset.
- Record the selected file path, season, and rationale in notes.
- State clearly that this is the dataset you will continue using through the rest of the learning path unless you intentionally restart the project.
- Define out-of-scope items for this week.

## Acceptance Criteria
- Dataset choice is explicit and documented.
- The exact source file path for later milestones is documented.
- The learner can explain that this dataset will carry into M2.
- Scope is narrow enough to be completed in one week.
- Out-of-scope items are listed.

## Verification
- Manual review of milestone notes and selected source links.
- Confirm selected file is reachable and downloadable.

## Notes
Keep the first scope intentionally small. Breadth can be added in later milestones.
`BRA.csv` is a teacher example, not the required course dataset.
The required dataset for later milestones is the one the learner chooses and records here.
